# 프로젝트 유형별 포트폴리오 템플릿

프로젝트 성격에 따라 강조할 구조를 다르게 선택한다.

## 제품/서비스 개선

- 기존 문제
- 사용자/고객 영향
- 개선 목표
- 가설 또는 판단 기준
- 개선 액션
- 전/후 변화
- 다음 개선점

## 운영 프로세스 개선

- 기존 운영 흐름
- 병목 또는 반복 이슈
- 개선한 프로세스
- 사용한 도구/문서화 방식
- 이해관계자 정렬 방식
- 시간/비용/오류/커뮤니케이션 개선

## 데이터 분석/실험

- 분석 질문
- 데이터 출처와 기간
- 분석 방법
- 발견한 인사이트
- 의사결정 또는 실행으로 연결한 내용
- 성과 또는 후속 실험

## 신규 기능/서비스 출시

- 출시 배경
- 목표 사용자와 문제
- 요구사항 정의
- 협업과 의사결정
- 출시 과정
- 출시 결과
- 배운 점

## 마케팅/캠페인

- 캠페인 목표
- 타깃과 메시지
- 채널/소재/예산 운영
- 실험과 최적화
- 성과 지표
- 다음 액션

## 협업/의사결정 개선

- 이해관계자와 갈등/비효율
- 조율한 기준
- 회의체/문서/프로세스
- 의사결정 속도 또는 품질 변화
- 남은 제약과 개선점
