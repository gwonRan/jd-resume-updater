# 포트폴리오 시각화 가이드

포트폴리오의 시각화는 예쁘게 보이는 장식이 아니라, 채용자가 핵심을 빠르게 이해하도록 돕는 증거다.

## 추천 시각화

| 상황 | 시각화 | 보여줄 것 |
|---|---|---|
| 전/후 변화가 있음 | Before/After 표 또는 막대그래프 | 개선율, 처리 시간, 오류율, 전환율 |
| 일정 관리가 중요함 | 타임라인 | 기간, 마감일, 주요 의사결정, 출시일 |
| 사용자 흐름 개선 | 퍼널 | 이탈 구간, 전환 개선, 병목 |
| 운영 프로세스 개선 | 프로세스 다이어그램 | 기존 흐름 vs 개선 후 흐름 |
| 협업이 중요함 | 이해관계자 맵 | 팀, 역할, 조율 범위 |
| 실험/분석 프로젝트 | 가설-실험-결과 표 | 분석 질문, 방법, 액션, 결과 |

## 첫 페이지 요약 템플릿

```markdown
# [프로젝트명]

한 줄 요약:
문제:
내 역할:
핵심 액션:
결과:
사용 스킬/도구:
왜 이 프로젝트가 중요한가:
```

## 주의

- 수치가 확인되지 않았으면 그래프로 만들지 않는다.
- 내부 지표나 고객 정보는 범위 또는 비율로 일반화한다.
- 한 화면에는 하나의 메시지만 담는다.
