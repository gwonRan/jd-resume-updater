# 포트폴리오 케이스 스터디 프레임워크

## 기본 구조

1. 프로젝트 요약
2. 문제 정의
3. 목표와 성공 기준
4. 내 역할과 오너십
5. 해결 과정
6. 사용한 스킬/도구
7. 협업과 의사결정
8. 결과
9. 회고와 다음 개선점
10. JD와 연결되는 역량

## 작성 기준

- 첫 화면에서 프로젝트 목적, 내 역할, 결과가 보여야 한다.
- "무엇을 만들었다"보다 "왜 만들었고 무엇이 달라졌는가"를 먼저 보여준다.
- 결과물 이미지만 나열하지 말고 판단 과정과 제약 조건을 설명한다.
- 스킬/도구는 사용 맥락과 연결한다.
- 성과 수치가 없으면 정성 변화와 후속 질문을 함께 제시한다.

## 프로젝트 요약 형식

```markdown
### [프로젝트명]

- 기간:
- 역할:
- 문제:
- 목표:
- 핵심 기여:
- 사용 스킬/도구:
- 결과:
- JD 연결 역량:
```
