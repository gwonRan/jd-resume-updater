# 마스터 이력서 프레임워크

## 목적

마스터 이력서는 제출용 완성본이 아니라 모든 경력 재료를 구조화해 둔 원본이다. JD별 제출본은 이 마스터에서 관련 경험만 골라 만든다.

## 수집 항목

- 회사/조직
- 직무/역할
- 재직기간
- 프로젝트명 또는 경험명
- 프로젝트 기간
- 마감일/출시일/운영 기간
- 문제 상황
- 목표
- 내 역할과 오너십 범위
- 구체적 행동
- 사용한 스킬, 도구, 방법론
- 스킬/도구를 사용한 맥락
- 협업 대상
- 산출물
- 정량 성과
- 정성 성과
- 배운 점/회고
- 공개 가능 여부
- 관련 JD 키워드

## JD별 제출본 만들기

1. JD에서 핵심 요구사항을 추출한다.
2. 마스터 재료에서 요구사항과 직접 연결되는 경험을 고른다.
3. 관련성이 낮은 경험은 삭제하거나 축소한다.
4. 성과가 강한 프로젝트를 앞에 배치한다.
5. 유사 직무군은 하나의 버전으로 묶고, 다른 직무군은 별도 버전으로 만든다.

## 성과 언어 변환

약한 표현:

`회의록 작성 및 일정 관리`

성과 언어:

`회의 결정사항과 일정 리스크를 문서화해 유관 부서의 실행 기준을 정렬하고 프로젝트 진행 병목을 조기 파악`

수치가 확인된 경우:

`Jira 기반 업무 진행률 트래킹을 도입해 일정 지연을 20% 감소`

## 스킬/도구 작성

- Skills 섹션에는 목록형으로 정리한다.
- Experience bullet에는 사용 맥락과 결과를 연결한다.
- 도구명만 나열하지 않는다.

예:

`SQL과 GA4를 활용해 사용자 행동 데이터를 분석하고, 전환 이탈 구간을 식별해 개선 과제를 정의`

## 기간/기한 작성

- `N개월간 [프로젝트/운영] 담당`
- `[출시일/마감일]에 맞춰 [산출물] 완료`
- `[기간] 동안 [지표]를 모니터링하고 개선`
- `[지연 기간] 지연된 프로젝트에서 [행동]으로 일정 리스크 완화`
