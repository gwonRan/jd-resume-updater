# 이력서 섹션별 작성 규칙

## Summary

- JD와 맞는 후보자 정체성을 2-4문장으로 요약한다.
- 경력 전체를 나열하지 말고 목표 직무에 맞는 강점만 선택한다.
- 수치가 있으면 1개 정도만 사용한다.
- 과장된 형용사보다 역할, 도메인, 강점, 성과를 보여준다.

## Core Skills

- 도구보다 역량 중심으로 묶고, 도구는 괄호 안에 보조로 넣는다.
- JD 핵심 키워드와 실제 사용 경험이 연결되는 스킬을 우선 배치한다.
- 사용한 적 없는 도구는 넣지 않는다.

예:

- 데이터 기반 문제 정의 및 성과 분석(SQL, GA4)
- 크로스펑셔널 프로젝트 관리(Jira, Confluence)
- 운영 프로세스 개선 및 문서화(Notion, Slack)

## Experience

- 회사별로 가장 JD와 관련 높은 bullet을 앞에 둔다.
- 각 bullet은 기여, 방법, 결과 중 최소 2개 이상을 포함한다.
- 같은 프로젝트의 세부 업무를 여러 bullet로 쪼개기보다 핵심 성과 중심으로 묶는다.

## Selected Projects

- JD와 직접 연결되는 프로젝트 2-4개를 선택한다.
- 프로젝트마다 문제, 역할, 결과가 드러나야 한다.
- 포트폴리오가 필요한 수준의 상세 설명은 portfolio-case-builder로 넘긴다.

## 경력기술서

- 이력서보다 맥락과 범위를 조금 더 설명한다.
- 프로젝트 기간, 조직/팀 규모, 이해관계자, 의사결정 맥락을 추가한다.
- 단, 내부 기밀과 민감 수치는 일반화한다.
