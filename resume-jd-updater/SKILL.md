# Resume JD Updater

## 목적

사용자가 제공한 일부 자료만으로도 목표 JD에 맞는 이력서와 경력기술서 콘텐츠를 만든다. 완성된 문서가 없어도 프로젝트 메모, 회고, 기존 bullet, 기억 기반 요약에서 근거를 추출해 이력서 언어로 바꾼다.

## 기본 워크플로우

1. **입력 분류**
   - 목표 JD
   - 기존 이력서 또는 경력 요약
   - 직무 정보와 연차
   - 프로젝트 메모
   - 성과 수치
   - 사용한 스킬/도구
   - 기밀 또는 비공개 정보
   - 언어, 톤, 지원 포지션 같은 선호

2. **JD 분석**
   JD의 필수 역량, 우대 역량, 반복 키워드, 직무 레벨 신호, 비즈니스 문제를 추출한다.

   자세한 기준은 `references/jd_analysis_framework.md`를 사용한다.

3. **마스터 이력서 재료 추출**
   모든 경험을 먼저 마스터 재료로 구조화한다. 이후 JD에 맞는 경험만 골라 제출본으로 축약한다.

   사용자가 마스터 이력서, 성과 언어, 스킬/도구, 기간/기한 정리를 요청하면 `references/master_resume_framework.md`를 사용한다.

4. **성과 언어 변환**
   단순 활동을 `문제/목표 + 행동 + 방법/도구 + 결과` 구조로 바꾼다. 수치가 없으면 임의 생성하지 않고 정성 성과 버전으로 쓴다.

   bullet 작성 기준은 `references/resume_bullet_patterns.md`를 사용한다.

5. **이력서 업데이트**
   기존 이력서가 있으면 전체를 갈아엎기보다 Summary, Skills, Experience, Selected Projects 중 필요한 섹션만 목적에 맞게 업데이트한다.

6. **품질 검토**
   근거성, JD 적합성, ATS 키워드, 직무 레벨, 기밀성, 개인 기여와 팀 성과 구분을 확인한다.

   기준은 `references/resume_quality_rubric.md`를 사용한다.

7. **후속 질문**
   부족한 정보는 `[확인 필요]`로 표시하거나 짧은 질문으로 묻는다. 질문 패턴은 `references/followup_questions.md`를 사용한다.

## 작성 원칙

- 사용자가 제공한 정보만 확정 사실로 사용한다.
- 수치, 기간, 도구, 프로젝트 범위, 오너십, 성과를 임의로 만들지 않는다.
- JD 키워드는 자연스럽게 반영하되 반복 삽입하지 않는다.
- 관련성이 낮은 경험은 줄이거나 제외한다.
- 개인 기여와 팀/회사 성과를 구분한다.
- 내부 시스템명, 고객명, 미공개 제품명, 민감 수치는 일반화한다.
- 사용자의 실제 연차와 역할 범위를 과장하지 않는다.

## 출력 모드

- JD 분석
- JD-경험 매칭표
- 마스터 이력서 재료집
- 성과 언어 변환표
- 스킬/도구 맥락표
- 이력서 bullet 후보
- Summary / Core Skills / Experience 섹션 업데이트
- 한국어/영문 이력서 버전
- 경력기술서 문장
- 면접용 STAR 답변 재료
- 후속 질문 리스트

## 입력 템플릿

사용자가 어떤 자료를 제공해야 하는지 묻는다면 `assets/input_template_ko.md`를 제공한다. 모든 항목을 채울 필요는 없다고 안내한다.
