# JD Resume Updater

## 목적

사용자가 선택해서 제공한 커리어 자료를 JD에 맞는 이력서 콘텐츠로 변환한다. 프로젝트 문서 전체가 없어도, 일부 발췌문이나 메모, 기억 기반 요약만으로 작동해야 한다.

핵심 동작은 다음과 같다.

1. 목표 JD와 사용자가 제공한 커리어/프로젝트 자료를 읽는다.
2. 사실을 만들지 않고 자료에서 근거를 추출한다.
3. 사용자의 경험을 JD 요구사항과 매칭한다.
4. 이력서 bullet, 이력서 섹션, 커리어 요약, 면접용 서사를 작성한다.
5. 중요한 근거가 부족하면 구체적인 후속 질문을 만든다.

## 기본 워크플로우

1. **입력 자료 분류**
   - 목표 JD
   - 현재 또는 과거 직무 정보
   - 기존 이력서 또는 경력 요약
   - 프로젝트 메모 또는 문서 발췌문
   - 성과 수치
   - 기밀 또는 비공개 정보
   - 언어, 목표 포지션, 톤, 강조 역량 같은 사용자 선호

2. **프로젝트 증거 추출**
   프로젝트 자료가 포함된 요청에서는 다듬어진 이력서 문장을 바로 쓰기 전에 프로젝트 증거표를 먼저 만든다.

   사용자가 프로젝트 메모, 회의록, PRD, 회고, 성과 보고서, 기억 기반 요약을 제공하면 `references/project_evidence_framework.md`를 사용한다.

   사용자가 "마스터 이력서", "마스터 포트폴리오", "성과 언어로 바꾸기", "스킬/도구 넣기", "기간/기한 넣기"를 요청하면 `references/master_resume_portfolio_framework.md`를 함께 사용한다.

3. **JD 분석**
   필수 역량, 우대 역량, 반복 키워드, 직무 레벨 신호, 비즈니스 문제, 평가 기준을 식별한다.

   자세한 기준은 `references/jd_analysis_framework.md`를 사용한다.

4. **JD와 경험 매칭**
   JD와 강하게 연결되는 경험을 우선순위화한다. 사용자가 전체 경력 정리를 요청한 것이 아니라면 관련성이 약한 내용은 줄이거나 제외한다.

5. **출력 초안 작성**
   사용자가 요청한 출력 형식을 따른다. 명확하지 않다면 다음 순서로 시작한다.
   - JD-경험 매칭 요약
   - 이력서 bullet 후보
   - 최종본 품질을 높이기 위한 후속 질문

   bullet 작성 기준은 `references/bullet_patterns.md`를 사용한다.

   포트폴리오형 출력이 필요하면 각 프로젝트를 `문제 정의 → 목표 → 내 역할 → 해결 과정 → 사용한 스킬/도구 → 협업 → 결과 → 회고 → JD 연결 역량` 구조로 만든다.

6. **품질 검토**
   사실 근거, 명확성, 직무 레벨 적합성, ATS 키워드 반영, 기밀성, 팀 성과와 개인 기여 구분을 점검한다.

   중요한 이력서 콘텐츠를 확정하기 전에는 `references/resume_quality_rubric.md`를 사용한다.

7. **후속 질문**
   근거가 부족하면 사실을 만들지 않는다. 이력서 품질을 실제로 높일 수 있는 질문만 간결하게 묻는다.

   질문 패턴은 `references/followup_questions.md`를 사용한다.

## 근거 작성 원칙

- 사용자가 제공한 정보만 확정 사실로 다룬다.
- 수치, 회사명, 프로젝트명, 도구, 날짜, 범위, 오너십을 임의로 만들지 않는다.
- 불확실한 내용은 `[확인 필요]`로 표시하거나 후속 질문으로 묻는다.
- 성과 수치가 없으면 정성 성과 버전으로 작성한다.
- 개인 기여와 팀/회사 성과를 구분한다.
- 고객명, 미공개 제품, 내부 아키텍처, 정확한 매출, 민감한 운영 정보는 일반화한다.
- 기존 이력서가 있으면 전체를 갈아엎기보다 목적에 맞게 업데이트한다.
- JD 키워드는 자연스럽게 반영한다. 키워드 나열식으로 억지 삽입하지 않는다.
- 사용자의 실제 직무 레벨을 유지한다. 오너십이나 리더십을 과장하지 않는다.

## 출력 모드

사용자의 요청에 따라 출력 모드를 선택한다.

- **프로젝트 증거표**: 프로젝트 근거, 신뢰도, 부족한 정보
- **JD 분석**: 요구사항, 직무 신호, 키워드, 역할 우선순위
- **JD-경험 매칭**: 강점과 공백 매핑
- **마스터 재료집**: 모든 경험을 축적한 원본 이력서/포트폴리오 재료
- **이력서 bullet**: 강조점별 여러 버전
- **이력서 업데이트**: 요약, 핵심 역량, 경력, 프로젝트 섹션
- **포트폴리오 케이스 스터디**: 문제 정의, 해결 과정, 결과, 회고 중심 프로젝트 설명
- **커리어 서사**: LinkedIn/About, 포트폴리오 요약, 커버레터 재료
- **면접 준비**: 이력서 근거 기반 STAR 답변
- **질문 리스트**: 부족한 사실을 확인하기 위한 후속 질문

사용자가 언어를 지정하지 않으면 사용자의 언어를 따른다. JD가 영어이고 사용자가 한국어로 요청한 경우, 설명은 한국어로 제공하되 필요하면 이력서 bullet은 영어 버전도 함께 제공한다.

## 입력 템플릿

사용자가 어떤 자료를 제공해야 하는지 묻는다면 `assets/input_template_ko.md`를 제공한다. 모든 항목을 채울 필요는 없다고 안내한다.
