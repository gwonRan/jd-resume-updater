# 이력서 품질 검토 기준

중요한 이력서 콘텐츠를 최종화하기 전에 사용한다.

## 점검 항목

- **근거성**: 모든 주장이 사용자 제공 자료로 뒷받침되거나 확인 필요로 표시되어 있는가?
- **구체성**: bullet에 범위, 방법, 이해관계자, 결과 중 하나 이상이 포함되어 있는가?
- **관련성**: 가장 강한 내용이 목표 JD와 연결되어 있는가?
- **수치 활용**: 수치는 제공되었거나 명시적으로 확인된 경우에만 사용했는가?
- **가독성**: bullet이 짧고 읽기 쉬우며 절이 과하게 많지 않은가?
- **직무 레벨 적합성**: 사용자의 실제 오너십 수준과 톤이 맞는가?
- **ATS 대응**: 중요한 JD 키워드가 자연스럽게 들어갔는가?
- **기밀성**: 민감한 정보가 익명화 또는 일반화되었는가?
- **기여 구분**: 개인 기여와 팀/회사 성과가 구분되어 있는가?
- **반복 방지**: 동사와 문장 구조가 지나치게 반복되지 않는가?

## 자주 필요한 수정

- bullet이 모호하면 범위, 방법, 이해관계자, 결과를 추가한다.
- 표현이 과장되어 보이면 동사를 낮추거나 확인 질문을 만든다.
- 임팩트가 약하면 정성 성과를 추가하거나 수치를 질문한다.
- JD 문장을 너무 그대로 따라 했으면 사용자의 실제 경험 언어로 다시 쓴다.
- 민감 정보가 들어가 있으면 고객, 제품, 데이터, 금액, 아키텍처를 일반화한다.

## 최종 검토 요약

중요한 출력물에는 짧게 다음을 덧붙인다.

- 가장 강한 JD 매칭 포인트
- 남아 있는 정보 공백
- `[확인 필요]`로 표시한 주장
