# 이력서 Bullet 작성 패턴

근거 기반 프로젝트 정보를 이력서 bullet로 바꿀 때 사용한다.

## 좋은 Bullet의 기본 공식

`행동 + 범위/맥락 + 방법/도구 + 결과`

또는:

`문제/목표 + 사용자의 기여 + 정량 또는 정성 성과`

## 패턴 유형

### 성과 중심

명확한 수치가 있을 때 사용한다.

`[범위]에서 [행동/방법]을 통해 [지표/프로세스]를 [결과]만큼 개선`

### 오너십 중심

업무 범위나 책임이 중요할 때 사용한다.

`[업무 영역/시스템/프로세스]를 오너십 있게 담당하고, [이해관계자]와 협업해 [산출물/결과]를 달성`

### 문제 해결 중심

프로젝트에 모호성, 제약, 복잡한 문제가 있었을 때 사용한다.

`[문제]를 [분석/구현/의사결정]으로 해결해 [비즈니스/사용자/팀 성과]를 가능하게 함`

### 협업 중심

JD에서 cross-functional collaboration이 중요할 때 사용한다.

`[팀/이해관계자]와 협업해 [산출물]을 만들고, [요구사항/트레이드오프]를 조율해 [결과]에 기여`

### 수치가 없는 버전

성과 수치가 없을 때 사용한다.

`[비즈니스/사용자 목표]를 지원하기 위해 [산출물/프로세스]를 구축/개선/정립하고, [팀/고객/운영]의 실행 기반을 강화`

## 한국어 Bullet 작성 기준

한국어 이력서 bullet은 구체적이고 간결해야 한다.

- 배경 설명보다 기여를 먼저 쓴다.
- 오너십이 드러나는 명사와 동사를 사용한다.
- 근거 없이 "혁신적으로", "압도적으로" 같은 표현을 쓰지 않는다.
- "기여", "개선", "설계", "운영", "자동화", "정립", "고도화", "리드"는 근거가 있을 때만 사용한다.

## 버전 생성 전략

중요한 프로젝트는 2-3가지 버전으로 작성한다.

- 비즈니스 임팩트 강조
- 기술 또는 프로세스 깊이 강조
- 리더십 또는 협업 강조
