# 프로젝트 증거 추출 프레임워크

사용자가 선택한 프로젝트 문서, 발췌문, 메모, 기억 기반 요약을 제공했을 때 사용한다.

## 추출 항목

각 프로젝트에서 다음 정보를 추출한다.

- 프로젝트명 또는 임시 이름
- 기간, 알려진 경우
- 마감일, 출시일, 운영 기간, 지연/단축 같은 시간 정보
- 비즈니스 또는 사용자 문제
- 프로젝트 목표
- 사용자의 역할
- 사용자의 구체적 기여
- 사용자가 내린 결정
- 사용한 도구, 방법론, 기술
- 이해관계자와 협업 맥락
- 제약조건 또는 복잡도
- 산출물
- 정량 성과
- 정성 성과
- 성과 언어로 바꿀 수 있는 원래 활동 표현
- 스킬/도구의 사용 맥락과 그로 가능해진 결과
- 마스터 이력서/포트폴리오에 넣을 가치가 있는지
- 포트폴리오 케이스 스터디로 확장할 가치가 있는지
- JD와 연결되는 역량
- 일반화해야 할 기밀 정보
- 부족하거나 불확실한 정보

## 신뢰도 라벨

다음 라벨을 사용한다.

- **확정**: 제공된 자료에서 직접 확인됨
- **확인 필요**: 그럴듯하지만 명시되어 있지 않음
- **부족**: 더 강한 이력서 bullet을 쓰려면 필요한 정보
- **민감**: 익명화 또는 일반화가 필요할 수 있는 정보

## 증거표 형식

필요하면 다음 형식을 사용한다.

| 프로젝트 | 근거 | 신뢰도 | JD 관련성 | 이력서 활용 |
|---|---|---|---|---|

## 확장 추출

마스터 이력서/포트폴리오나 포트폴리오 케이스 스터디를 만들 때는 `references/master_resume_portfolio_framework.md`를 함께 사용한다.

## 중요 원칙

모호한 팀 단위 성과를 개인 성과로 바꾸지 않는다. 자료에 팀이 무언가를 달성했다고 되어 있으면, 사용자의 개인 기여가 명확할 때만 개인 성과로 작성한다.
