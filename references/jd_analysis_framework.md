# JD 분석 프레임워크

목표 JD를 분석할 때 사용한다.

## 추출할 정보

- 직무명과 예상 직무 레벨
- 회사 또는 팀의 미션 신호
- 핵심 책임
- 필수 역량
- 우대 역량
- 도메인 지식
- 도구, 방법론, 업무 프로세스 기대사항
- 반복 등장하는 키워드와 표현
- 이 역할이 해결해야 하는 비즈니스 문제
- 협업 및 이해관계자 관리 기대사항
- 리더십, 오너십, 실행력 관련 신호

## 요구사항 분류

JD 요구사항을 다음 그룹으로 나눈다.

- **반드시 보여줘야 함**: 역할의 중심이며 서류 평가에서 중요할 가능성이 높음
- **보여주면 좋음**: 차별화 요소가 될 수 있음
- **가볍게 언급 가능**: 도움이 되지만 핵심은 아님
- **공백 또는 리스크**: 사용자가 제공한 근거로는 아직 뒷받침되지 않음

## 포지셔닝 도출

이 JD에 맞게 후보자를 어떻게 포지셔닝할지 요약한다.

- 핵심 셀링 포인트
- 이를 뒷받침하는 강점 2-4개
- 가장 관련성 높은 프로젝트
- 전면에 둘 역량
- 줄이거나 뒤로 뺄 역량
- 정직하게 보완해야 할 공백

## 출력 형식

필요하면 다음과 같은 간결한 표를 사용한다.

| JD 신호 | 중요도 | 매칭되는 경험 근거 | 이력서 반영 방향 |
|---|---:|---|---|

JD의 모든 표현을 기계적으로 따라 하지 않는다. 사용자의 실제 경험 언어로 자연스럽게 번역한다.
